SPOTIFY-ACCESSIBILITY — WHAT WE BUILT (FEATURE & FILE INVENTORY)

Plain overview of the demo app: navigation, Axis mode, voice inputs, command
palette, pins, mock catalog, and supporting assets/scripts.

--------------------------------------------------------------------------------
1. APP SHELL & NAVIGATION
--------------------------------------------------------------------------------
• React + Vite + TypeScript SPA (src/main.tsx → src/ui/App.tsx).
• Bottom tabs: Home (Discover), Search, Your Library.
• Mini player strip; Now Playing sheet; Artist sheet; Command palette overlay.
• Profile menu slide-over (Axis toggle, “Axis introduction”, premium-style rows).
• Context menu sheet for landmarks / tracks (pin, play, etc.).
• Toasts for pin feedback; optional TTS (speak helper in tts.ts).
• Axis tutorial overlay; Axis settings persisted (axisSettings.ts).


--------------------------------------------------------------------------------
2. HOME — STANDARD MODE (Axis OFF)
--------------------------------------------------------------------------------
• Header: circular profile (your photo when asset exists), blue notify dot.
• Category pills: All, Music, Podcasts, Audiobooks.
• White search row: magnifying glass, text field, microphone = voice-to-text
  only (Web Speech API via useAxisSearchSpeech — does NOT open command palette).
• 2×4 grid of Spotify-style shortcuts (Daily Mix 1, On Repeat, Discover Weekly,
  Release Radar, Liked Songs, Chill Hits, Deep Focus, Party Starters).
  Thumbnails: public/home/tile-01.png … tile-08.png (picsum fallback if missing).
• “Try Axis” promo card (AxisEntryCard) → opens Axis tutorial.
• No top-row command-palette mic on this screen.

File: src/ui/screens/DiscoverScreen.tsx


--------------------------------------------------------------------------------
3. HOME — AXIS MODE (Axis ON)
--------------------------------------------------------------------------------
• Same top pattern: profile, main tabs Songs | Artists | Albums | Playlists,
  white search row, search mic = voice-to-text only.
• NO microphone in the top-right (command palette removed from header); open
  palette from scroll content instead.
• Recently played: 2×2 category tiles (Songs, Artists, Albums, Playlist) with
  colors + art (public/axis/axis-recent-*.png + fallbacks). Tap selects tab;
  long-press opens context.
• Axis controls card: “Commands & voice” → opens command palette.
• Pinned section (2 landmarks from session).
• Top Songs: sub-tabs All | Pop | Hip-hop; rows with play; ListRow + optional
  thumbs; uses mockData.axisHomeTopSongRows.

File: src/ui/screens/DiscoverScreen.tsx


--------------------------------------------------------------------------------
4. COMMAND PALETTE
--------------------------------------------------------------------------------
• Bottom sheet: Done, optional first-run hint, then primary voice control.
• “Find music & actions”: NO text typing field.
• Large green full-width button: “Say something” with microphone on the RIGHT.
  Uses Web Speech API; transcript fills internal query and drives
  filterPaletteSearch → Results list (play / long-press context).
• Pinned rows; Quick actions (like, add to queue, play liked songs, etc.).
• Escape closes; focus returns to [data-command-palette-trigger] opener.

File: src/ui/overlays/CommandPalette.tsx


--------------------------------------------------------------------------------
5. VOICE / SPEECH
--------------------------------------------------------------------------------
• useAxisSearchSpeech.ts — SpeechRecognition / webkitSpeechRecognition, final
  transcript callback, listening + supported state.
• src/speech-recognition.d.ts — ambient TS types so `tsc` passes (e.g. Vercel).
